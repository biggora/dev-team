# DEBUG

**1. Локальное обнаружение** — если запустить `claude` из директории самого плагина, он подхватится автоматически, потому что `.claude-plugin/plugin.json` лежит в текущей директории:

```bash
cd D:/Projects/dev-team
claude --debug
# Плагин уже доступен: /dev-team, /dev-team-node, /dev-team-python
```

Но это ограничено — вы работаете внутри директории плагина, а не целевого проекта.

**2. Установка + удаление** — на самом деле это быстро и чисто:

```bash
# Установить из локальной директории (ничего не копирует, создаёт ссылку)
claude plugins add D:/Projects/dev-team

# Работаете в любом проекте
cd D:/Projects/my-nestjs-app
claude --debug
/dev-team-node Добавить health check endpoint

# Когда закончили тестирование
claude plugins remove dev-team
```

`plugins add` с локальным путём работает через симлинк — мгновенная установка, мгновенное удаление, ничего не засоряет.

Для тестирования я бы использовал второй способ — так вы проверите плагин на реальном проекте, а не на самом себе.
