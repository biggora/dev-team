# Built-in Components

Next.js provides optimized components for common web patterns.

## Link Component

Use `<Link>` for client-side navigation:

```tsx
import Link from 'next/link'

export default function Navigation() {
  return (
    <nav>
      <Link href="/">Home</Link>
      <Link href="/about">About</Link>
    </nav>
  )
}
```

### Link Props

- `href`: Path or URL (required)
- `prefetch`: Enable/disable prefetching (default: `true`)
- `replace`: Replace current history entry
- `scroll`: Scroll to top on navigation (default: `true`)

```tsx
<Link href="/dashboard" prefetch={false} scroll={false}>
  Dashboard
</Link>
```

## Image Component

Optimized image component with automatic optimization:

```tsx
import Image from 'next/image'

export default function Page() {
  return (
    <Image
      src="/profile.png"
      width={500}
      height={500}
      alt="Profile picture"
    />
  )
}
```

### Image Props

- `src`: Image source (required)
- `alt`: Alt text (required)
- `width` / `height`: Dimensions (required unless using `fill`)
- `fill`: Fill parent container
- `sizes`: Responsive image sizes
- `quality`: Image quality (1-100, default: 75)
- `placeholder`: Placeholder type (`blur` or `empty`)
- `blurDataURL`: Base64 blur placeholder

```tsx
<Image
  src="/hero.jpg"
  fill
  sizes="(max-width: 768px) 100vw, 50vw"
  alt="Hero image"
  placeholder="blur"
  blurDataURL="data:image/jpeg;base64,..."
/>
```

### Remote Images

Configure remote images in `next.config.js`:

```js
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
        pathname: '/images/**',
      },
    ],
  },
}
```

## Script Component

Optimize third-party scripts:

```tsx
import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        strategy="afterInteractive"
      />
    </>
  )
}
```

### Script Strategies

- `beforeInteractive`: Load before page becomes interactive
- `afterInteractive`: Load after page becomes interactive (default)
- `lazyOnload`: Load during idle time
- `worker`: Load in a web worker

```tsx
<Script
  src="https://example.com/analytics.js"
  strategy="afterInteractive"
  onLoad={() => {
    // Script loaded
  }}
/>
```

## Font Component

Optimize font loading:

```tsx
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

### Font Options

- `subsets`: Font subsets to load
- `weight`: Font weights
- `style`: Font styles
- `variable`: CSS variable name
- `display`: Font display strategy

```tsx
import { Roboto } from 'next/font/google'

const roboto = Roboto({
  weight: ['400', '700'],
  subsets: ['latin'],
  variable: '--font-roboto',
})

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div className={roboto.variable}>
      {children}
    </div>
  )
}
```

## Form Component

Use `<Form>` from `next/form` for forms that navigate on submission (search forms, filter forms). It extends `<form>` with prefetching, client-side navigation, and progressive enhancement:

```tsx
import Form from 'next/form'

export default function SearchForm() {
  return (
    <Form action="/search">
      <input name="q" placeholder="Search..." />
      <button type="submit">Search</button>
    </Form>
  )
}
```

When submitted, the form navigates to `/search?q=...` using client-side navigation. The target page's `loading.tsx` is prefetched when the form enters the viewport.

For mutations (creating, updating, deleting data), use Server Actions with the standard `<form>` element instead.

## `server-only` Import Guard

Import `server-only` in modules that must never be imported from Client Components. This causes a build error if accidentally imported on the client, protecting secrets and database connections:

```ts
// app/lib/db.ts
import 'server-only'
import { Pool } from 'pg'

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
})

export async function getUsers() {
  const { rows } = await pool.query('SELECT * FROM users')
  return rows
}
```

Install with `npm install server-only`. Always use on modules that access databases, environment variables with secrets, or internal APIs.

## Key Points

- Use `<Link>` for all internal navigation
- Use `<Image>` for optimized images with automatic optimization
- Use `<Script>` for third-party scripts with loading strategies
- Use `next/font` for optimized, self-hosted font loading
- Use `<Form>` from `next/form` for navigation forms (search, filters)
- Use `import 'server-only'` on database and secret-containing modules
- Configure remote images in `next.config.js`
