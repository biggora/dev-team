# Cache Components and Partial Prerendering (PPR)

Cache Components let you cache any async computation — not just `fetch` calls. They use the `'use cache'` directive and replace `unstable_cache`. Combined with `cacheComponents: true` in next.config.js, they enable Partial Prerendering (PPR).

## Enabling Cache Components

```js
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  cacheComponents: true,
}

module.exports = nextConfig
```

This replaces the old `experimental.ppr` flag from Next.js 15 canaries.

## The `'use cache'` Directive

Add `'use cache'` to any async function to cache its result. Tag with `cacheTag` for on-demand invalidation:

```ts
import { cacheTag } from 'next/cache'

export async function getProducts() {
  'use cache'
  cacheTag('products')

  const products = await db.query('SELECT * FROM products')
  return products
}
```

The function's return value is cached and shared across all requests until the tag is invalidated. This works with any data source — databases, ORMs, file system, computations — not just `fetch`.

## Cache Variants

Next.js 16 has three cache directives for different scenarios:

### `'use cache'` (Default/Static)

Cached at build time. Shared across all users and requests. Best for data that rarely changes.

```ts
async function getProduct(id: string) {
  'use cache'
  cacheTag(`product-${id}`)

  return db.products.find({ where: { id } })
}
```

### `'use cache: remote'`

Cached at runtime in a shared remote handler. Use for data that is the same for all users but needs to be fetched at request time (not build time).

```ts
import { cacheLife, cacheTag } from 'next/cache'
import { connection } from 'next/server'

async function getProductPrice(id: string) {
  'use cache: remote'
  cacheTag(`product-price-${id}`)
  cacheLife({ expire: 300 }) // 5 minutes

  return db.products.getPrice({ where: { id } })
}
```

### `'use cache: private'`

Cached per-user. Never shared between users. Use for personalized data like recommendations, user settings, or cart contents.

```ts
import { cacheLife } from 'next/cache'
import { cookies } from 'next/headers'

async function getRecommendations(productId: string) {
  'use cache: private'
  cacheLife({ expire: 60 }) // 1 minute

  const sessionId = (await cookies()).get('session-id')?.value
  return db.recommendations.findMany({
    where: { productId, sessionId },
  })
}
```

## `cacheLife` — Fine-Grained Expiration

Control how long cached data lives:

```ts
import { cacheLife } from 'next/cache'

async function getFeedItems() {
  'use cache: remote'
  cacheLife({ expire: 120 }) // 2 minutes

  const response = await fetch('https://api.example.com/feed')
  return response.json()
}
```

You can also use named profiles:

```ts
// Predefined profiles
cacheLife('max')     // Long-lived cache
cacheLife('default') // Standard cache
cacheLife('min')     // Short-lived cache
```

## `connection()` — Defer to Request Time

Use `connection()` from `next/server` to explicitly defer rendering to request time. This is useful when you want part of the page to be static (prerendered) and part to be dynamic (streamed).

```tsx
import { Suspense } from 'react'
import { connection } from 'next/server'
import { cacheLife, cacheTag } from 'next/cache'

export default async function FeedPage() {
  return (
    <div>
      <h1>Feed</h1> {/* Static — prerendered */}
      <Suspense fallback={<Skeleton />}>
        <FeedItems /> {/* Dynamic — streamed */}
      </Suspense>
    </div>
  )
}

async function FeedItems() {
  await connection() // Defer to request time

  const items = await getFeedItems()
  return items.map((item) => <FeedItem key={item.id} item={item} />)
}

async function getFeedItems() {
  'use cache: remote'
  cacheTag('feed-items')
  cacheLife({ expire: 120 })

  const response = await fetch('https://api.example.com/feed')
  return response.json()
}
```

## Mixed Caching Strategies

Combine all three cache variants in a single page for optimal performance:

```tsx
import { Suspense } from 'react'
import { connection } from 'next/server'
import { cacheLife, cacheTag } from 'next/cache'

// Static: prerendered at build time, shared across all users
async function getProduct(id: string) {
  'use cache'
  cacheTag(`product-${id}`)
  return db.products.find({ where: { id } })
}

// Remote: cached at runtime, shared across all users, expires in 5 min
async function getProductPrice(id: string) {
  'use cache: remote'
  cacheTag(`product-price-${id}`)
  cacheLife({ expire: 300 })
  return db.products.getPrice({ where: { id } })
}

// Private: cached per-user, expires in 1 min
async function getRecommendations(productId: string) {
  'use cache: private'
  cacheLife({ expire: 60 })
  const sessionId = (await cookies()).get('session-id')?.value
  return db.recommendations.findMany({ where: { productId, sessionId } })
}

export default async function ProductPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const product = await getProduct(id) // Static

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.description}</p>

      <Suspense fallback={<PriceSkeleton />}>
        <PriceSection productId={id} /> {/* Shared dynamic */}
      </Suspense>

      <Suspense fallback={<RecommendationsSkeleton />}>
        <Recommendations productId={id} /> {/* Per-user dynamic */}
      </Suspense>
    </div>
  )
}

async function PriceSection({ productId }: { productId: string }) {
  await connection()
  const price = await getProductPrice(productId)
  return <div>Price: ${price}</div>
}

async function Recommendations({ productId }: { productId: string }) {
  const recs = await getRecommendations(productId)
  return (
    <ul>
      {recs.map((r) => <li key={r.id}>{r.name}</li>)}
    </ul>
  )
}
```

## Invalidation: `updateTag` vs `revalidateTag`

| Function | Where | Behavior | Use case |
|----------|-------|----------|----------|
| `updateTag(tag)` | Server Actions only | Immediately expires cache. Next request gets fresh data. | Read-your-own-writes (user creates post, redirects, sees it immediately) |
| `revalidateTag(tag, 'max')` | Server Actions + Route Handlers | Stale-while-revalidate. Serves old data while fetching fresh in background. | Background updates (admin price change, users see update on next visit) |
| `revalidateTag(tag)` | Server Actions + Route Handlers | Legacy immediate expiration (deprecated). Prefer `'max'` profile. | Avoid — use `updateTag` or `revalidateTag(tag, 'max')` instead |

```ts
'use server'

import { updateTag } from 'next/cache'
import { revalidateTag } from 'next/cache'
import { redirect } from 'next/navigation'

// User creates a post — they need to see it immediately
export async function createPost(formData: FormData) {
  const post = await db.post.create({
    data: { title: formData.get('title'), content: formData.get('content') },
  })

  updateTag('posts')
  updateTag(`post-${post.id}`)
  redirect(`/posts/${post.id}`)
}

// Admin updates pricing — background revalidation is fine
export async function updatePricing() {
  await db.products.recalculatePrices()
  revalidateTag('products', 'max')
}
```

## Key Points

- Use `'use cache'` instead of `unstable_cache` (which is deprecated)
- Three variants: default (static/build-time), `remote` (shared runtime), `private` (per-user)
- `cacheLife({ expire: N })` controls expiration in seconds
- `connection()` from `next/server` defers rendering to request time
- Enable PPR with `cacheComponents: true` in next.config.js
- `updateTag` = immediate expiration (read-your-own-writes)
- `revalidateTag(tag, 'max')` = stale-while-revalidate (recommended for background updates)
