# Metadata and SEO

Next.js provides a Metadata API for defining page metadata (title, description, Open Graph, etc.) that is automatically rendered as HTML `<head>` elements.

## Static Metadata

Export a `metadata` object from a `layout.tsx` or `page.tsx`:

```tsx
// app/page.tsx
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'My App',
  description: 'A description of my app',
}

export default function Page() {
  return <div>Home</div>
}
```

## Dynamic Metadata with `generateMetadata`

Use `generateMetadata` for dynamic pages that need to fetch data for their metadata:

```tsx
// app/blog/[slug]/page.tsx
import type { Metadata } from 'next'
import { getPost } from '@/app/lib/api'

export async function generateMetadata({
  params,
}: {
  params: Promise<{ slug: string }>
}): Promise<Metadata> {
  const { slug } = await params
  const post = await getPost(slug)

  if (!post) {
    return { title: 'Post Not Found' }
  }

  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.coverImage],
    },
  }
}

export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await getPost(slug)
  // ...
}
```

The `fetch` call in `generateMetadata` is automatically deduplicated with the same call in the page component — the API is only hit once.

## Title Templates

Define title templates in layouts that child pages inherit:

```tsx
// app/layout.tsx
export const metadata: Metadata = {
  title: {
    template: '%s | My App',
    default: 'My App',
  },
}
```

```tsx
// app/blog/page.tsx
export const metadata: Metadata = {
  title: 'Blog', // Renders as "Blog | My App"
}
```

## Viewport

Export viewport configuration separately from metadata:

```tsx
import type { Viewport } from 'next'

export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  themeColor: '#000000',
}
```

## Open Graph and Twitter

```tsx
export const metadata: Metadata = {
  openGraph: {
    title: 'My App',
    description: 'A description of my app',
    url: 'https://example.com',
    siteName: 'My App',
    images: [
      {
        url: 'https://example.com/og.png',
        width: 1200,
        height: 630,
        alt: 'My App',
      },
    ],
    locale: 'en_US',
    type: 'website',
  },
  twitter: {
    card: 'summary_large_image',
    title: 'My App',
    description: 'A description of my app',
    images: ['https://example.com/og.png'],
  },
}
```

## JSON-LD Structured Data

Add JSON-LD structured data in page components. Note: this is safe because `JSON.stringify` produces valid JSON that cannot contain executable scripts:

```tsx
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await getPost(slug)

  const jsonLd = {
    '@context': 'https://schema.org',
    '@type': 'BlogPosting',
    headline: post.title,
    datePublished: post.publishedAt,
    author: {
      '@type': 'Person',
      name: post.author,
    },
  }

  return (
    <>
      <script
        type="application/ld+json"
        // Safe: JSON.stringify output cannot contain script injection
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      <article>
        <h1>{post.title}</h1>
        {/* ... */}
      </article>
    </>
  )
}
```

## Robots and Sitemap

```tsx
// app/robots.ts
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: '*',
      allow: '/',
      disallow: '/admin/',
    },
    sitemap: 'https://example.com/sitemap.xml',
  }
}
```

```tsx
// app/sitemap.ts
import type { MetadataRoute } from 'next'

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: 'https://example.com',
      lastModified: new Date(),
      changeFrequency: 'yearly',
      priority: 1,
    },
    {
      url: 'https://example.com/blog',
      lastModified: new Date(),
      changeFrequency: 'weekly',
      priority: 0.8,
    },
  ]
}
```

## Dynamic Sitemap

Generate sitemaps from data:

```tsx
// app/sitemap.ts
import type { MetadataRoute } from 'next'

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const posts = await fetch('https://api.example.com/posts').then((r) =>
    r.json()
  )

  const postEntries = posts.map((post: { slug: string; updatedAt: string }) => ({
    url: `https://example.com/blog/${post.slug}`,
    lastModified: new Date(post.updatedAt),
    changeFrequency: 'monthly' as const,
    priority: 0.7,
  }))

  return [
    { url: 'https://example.com', lastModified: new Date(), priority: 1 },
    ...postEntries,
  ]
}
```

## Key Points

- Use static `metadata` export for pages with known metadata
- Use `generateMetadata` for dynamic pages — `params` is a `Promise`, must be awaited
- `fetch` calls in `generateMetadata` are deduplicated with the page component
- Export `viewport` separately from `metadata`
- Title templates in layouts propagate to child pages
- Use `robots.ts` and `sitemap.ts` file conventions for SEO files
- Add JSON-LD structured data directly in page components using `JSON.stringify`
